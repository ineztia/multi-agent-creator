# 质量审查流程

> 本规则定义引擎的三级质量审查流程，供质量审查专家执行时参考。
> 审查标准参照 `../1-engine-design.md` §6 质量控制规范。

---

## 1. 审查总览

```
第一轮（结构审查）──→ 第二轮（内容审查）──→ 第三轮（综合评审）
      ↓ 不合格                ↓ 不合格                ↓
    打回修改              打回修改            标注遗留问题
```

**终止条件**：单份文件最多打回 2 次（共 3 次提交机会）。第 3 次提交后无论结果如何，标注遗留问题并继续流程。

## 2. 第一轮：结构完整性审查

**审查重点**：产出是否齐全、格式是否合规

**检查清单**：

- [ ] 所有维度研究区均有子专题报告文件
- [ ] 所有维度研究区均有综合评估文件
- [ ] 每份报告包含完整的章节结构（摘要、研究问题、方法、发现、分析、结论）
- [ ] 每个研究区的 `data/sources-registry.md` 已填写
- [ ] 文件命名符合 `1-engine-design.md` §2.1 的规范
- [ ] 扩展研究区（如有）报告已产出
- [ ] 综合判断区报告已产出

**产出文件**：`review-round1.md`

**格式**：

```markdown
# 第一轮审查报告 - 结构完整性

## 审查结果：通过 / 不通过

## 问题清单

| 编号 | 文件 | 问题描述 | 严重程度 | 责任专家 |
|------|------|---------|---------|---------|

## 审查意见
[整体评价和建议]
```

## 3. 第二轮：内容质量审查

**审查重点**：论证是否严密、数据是否可靠、结论是否合理

**检查清单**：

- [ ] 每个关键论断有明确的数据或文献支撑
- [ ] 数据来源权威性评级合理
- [ ] 论证过程逻辑严密，无跳跃性结论
- [ ] 结论有充分的证据支撑，无过度推断
- [ ] 同一维度内各子专题之间逻辑一致
- [ ] 综合评估准确反映了各子专题的研究发现
- [ ] 对争议性问题呈现了多方观点

**产出文件**：`review-round2.md`（格式同上）

## 4. 第三轮：综合评审

**审查重点**：整体一致性、创新性、实用价值

**检查清单**：

- [ ] 各维度研究之间逻辑一致，无相互矛盾
- [ ] 综合判断合理整合了各维度研究成果
- [ ] 核心研究问题得到了充分回答
- [ ] 研究具有独到见解和创新点
- [ ] 研究成果对目标读者有实用价值
- [ ] 整体报告结构完整、层次清晰
- [ ] 所有事实核查通过（参照 `fact-check-workflow.md`）

**产出文件**：`review-round3-综合报告.md`

**格式**：

```markdown
# 第三轮审查报告 - 综合评审

## 总体评价
[整体质量评估，200-300字]

## 各维度质量评分

| 维度 | 结构完整性 | 论证质量 | 数据可靠性 | 创新性 | 综合评分 |
|------|-----------|---------|-----------|-------|---------|

## 突出优点
[列举研究的亮点]

## 遗留问题
[列举尚存的问题及处理建议]

## 审查结论
通过 / 有条件通过（附遗留问题清单）
```

## 5. 打回修改流程

当审查不通过时：

1. 在审查报告中明确指出问题所在及对应的责任专家
2. 将审查报告发送给责任专家
3. 责任专家针对指出的问题进行修改
4. 修改后重新提交审查
5. 审查者仅复核修改部分，无需全量重审
