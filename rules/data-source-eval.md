# 数据源评估参考

> 本文件是 L2 通用规范模板，供生成器构造引擎实例时参考。生成器应基于此模板为每个具体任务动态生成 `{TASK}/rules/` 中的任务专属数据源评估规范，并融入该领域的权威数据源清单。

---

## 1. 权威性评级标准

| 评级 | 定义 | 典型来源 |
|------|------|---------|
| **高** | 经同行评审或权威机构发布，数据采集方法透明 | 顶级学术期刊、政府官方统计、国际组织报告（世界银行/IMF/WHO 等）、行业权威机构年报 |
| **中** | 来源可信但未经严格同行评审，或存在潜在利益偏向 | 知名咨询公司报告（麦肯锡/BCG 等）、行业协会数据、企业财报、主流媒体深度报道 |
| **低** | 来源不明、数据采集方法不透明、或存在明显偏向 | 个人博客、社交媒体、匿名来源、利益相关方的自我宣传材料 |

---

## 2. 评级规则

- 默认使用高权威性来源，中权威性来源作为补充
- 低权威性来源仅在无替代来源时使用，且必须标注「来源权威性较低」
- 同一结论尽量由 2 个以上独立来源交叉验证

---

## 3. 数据时效性要求

| 数据类型 | 推荐时效 | 说明 |
|---------|---------|------|
| 统计数据 | 最近 2 年 | 如只有更早数据，需标注「最新可得数据为 XXXX 年」 |
| 行业报告 | 最近 3 年 | 快速变化领域建议 1 年内 |
| 学术论文 | 最近 5 年 | 经典文献不受此限 |
| 政策法规 | 现行有效 | 需确认是否已被修订或废止 |

---

## 4. 数据使用与论证的关系

数据源评估不仅是来源信度判断，更需要注意以下与论证质量相关的要求：

- **实证与推测的区分**：实证获取的数据与估算推测必须明确标注。推测值不得以确定性格式（如高权威性评级的表格）呈现，否则构成虚假精确
- **数据的论证嵌入**：引用数据时必须将其嵌入推理链（数据→意味着→因为→所以），而非仅仅列出数据
- **数据局限性说明**：对关键数据应说明其可能的偏差来源、样本局限、时效性限制等

---

## 5. 数据源注册表规范

每个维度研究区下应维护 `data/sources-registry.md`，记录该维度研究所使用的所有数据源：

| 字段 | 说明 |
|------|------|
| 来源名称 | 数据源的完整名称 |
| 类型 | 学术论文/行业报告/政府数据/新闻报道等 |
| 权威性评级 | 高/中/低（依据本文件 §1 标准） |
| 访问时间 | 数据获取时间 |
| 使用位置 | 在哪份研究报告中引用 |
| 备注 | 数据局限性或特殊说明 |
