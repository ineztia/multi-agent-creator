# 研究报告写作规范

> 本规则供引擎执行 AI 在撰写研究报告时参考，适用于所有层级的产出文件（子专题报告、维度综合评估、最终报告）。

---

## 1. 总体要求

- 以学术期刊论文的标准撰写，不是提纲、不是大纲、不是要点罗列
- 每份报告应是**可独立阅读的完整文章**，包含完整的论证过程和结论
- 语言风格：严谨、客观、专业，避免口语化和主观臆断

## 2. 子专题研究报告结构

```markdown
# {子专题名称}

## 摘要
[200-300字，概述研究问题、方法、关键发现和结论]

## 1. 研究问题
[明确界定本子专题要回答的核心问题]

## 2. 研究方法与数据来源
[说明采用的研究方法、数据来源及其权威性评级]

## 3. 研究发现
### 3.1 [发现1]
### 3.2 [发现2]
### 3.N [发现N]
[每个发现需有数据支撑和论证过程]

## 4. 分析与讨论
[对研究发现的深度分析，揭示规律、因果关系、影响因素]

## 5. 结论与启示
[明确回答研究问题，提炼关键洞察]

## 参考来源
[列出所有引用的数据源，格式参考 data/sources-registry.md]
```

## 3. 维度综合评估结构

```markdown
# {维度名称} - 综合评估

## 摘要
[300-500字，概述该维度的整体研究发现]

## 1. 维度概述
[该维度的研究范围、核心问题、包含的子专题]

## 2. 各子专题核心发现
### 2.1 {子专题1} 核心发现
### 2.2 {子专题2} 核心发现
[简要提炼每个子专题的关键结论]

## 3. 跨子专题综合分析
[整合各子专题发现，识别共性规律、矛盾点、互补关系]

## 4. 该维度整体判断
[形成该维度的整体评估结论]

## 5. 对核心问题的贡献
[该维度的研究结果如何回应事项的核心研究问题]
```

## 4. 写作规范细则

### 4.1 论证规范

- **有据可查**：每个关键论断必须有明确的数据或文献支撑
- **标注来源**：引用数据时标注来源（如：「据麦肯锡 2024 报告...」）
- **区分事实与观点**：事实性陈述与分析性判断需明确区分
- **量化优先**：能用数据量化的结论优先使用数据，避免纯定性描述

### 4.2 格式规范

- 使用 Markdown 格式
- 标题层级不超过 4 级（`#` ~ `####`）
- 关键数据使用表格呈现
- 重要对比使用对照表
- 每节控制在合理长度，避免单节过长

### 4.3 学术规范

- 不抄袭、不编造数据
- 明确标注数据的获取时间和来源
- 对争议性结论呈现多方观点
- 承认研究局限性
