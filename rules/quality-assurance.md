# 质量保证参考

> 本文件是生成器构造引擎实例时的质量保证参考模板。生成器在构造 `{TASK}/rules/` 时，基于本模板适配具体事项的质量控制需求。

---

## 1. 质量属性

引擎产出的研究成果必须满足以下四项质量属性：

| 质量属性 | 标准 |
|---------|------|
| 学术规范性 | 研究报告严格符合学术规范，是完整文章而非提纲 |
| 逻辑完整性 | 研讨过程层次清晰、论证严密 |
| 数据可靠性 | 数据来源权威，事实准确 |
| 创新性 | 有独到见解和创新点 |

---

## 2. 三级审查制度

质量审查由质量审查专家执行，分三轮递进审查：

| 轮次 | 审查重点 | 产出文件 |
|------|---------|---------|
| 第一轮 | **结构完整性**：各阶段产出是否齐全、格式是否合规 | `review-round1.md` |
| 第二轮 | **内容质量**：论证是否严密、数据是否可靠、结论是否合理 | `review-round2.md` |
| 第三轮 | **综合评审**：整体一致性、创新性、实用价值 | `review-round3-综合报告.md` |

### 2.1 第一轮审查清单（结构完整性）

- [ ] 所有维度的子专题研究报告是否齐全
- [ ] 所有维度综合评估是否齐全
- [ ] 扩展研究报告（如有）是否齐全
- [ ] 综合判断报告是否齐全
- [ ] 每份报告的章节结构是否符合写作规范
- [ ] 每份报告的篇幅是否满足内容实质性标准（2000 字正文）
- [ ] 数据源注册表是否已维护

### 2.2 第二轮审查清单（内容质量）

- [ ] 每个关键论断是否有明确的数据或文献支撑
- [ ] 数据来源是否标注且可查
- [ ] 事实性陈述与分析性判断是否明确区分
- [ ] 维度综合评估是否逐一引用了每份子专题的具体发现
- [ ] 综合判断是否实际整合了前序阶段的具体数据和结论（而非泛泛概括）
- [ ] 不同维度的研究是否存在矛盾结论（若有，是否已在综合判断中说明）
- [ ] 数据来源的权威性评级是否准确

### 2.3 第三轮审查清单（综合评审）

- [ ] 各维度研究与核心问题的关联是否清晰
- [ ] 维度间是否存在重复研究或遗漏视角
- [ ] 研究是否有独到见解和创新点（而非仅综述现有观点）
- [ ] 研究结论是否具有实用价值和可操作性
- [ ] 整体叙事是否连贯一致
- [ ] 研究局限性是否已客观标注

---

## 3. 事实核查

### 3.1 核查范围

- 核查数据来源的权威性和时效性
- 核查关键事实的准确性
- 核查引用数据是否与原始来源一致

### 3.2 核查流程

1. 逐份报告扫描关键数据点和事实性陈述
2. 对每个数据点验证：来源是否存在、数据是否准确引用、是否为最新可得数据
3. 标记无法验证的数据点，评估其对结论的影响
4. 记录核查过程和结果到 `fact-check-log.md`

### 3.3 fact-check-log.md 格式

```markdown
# 事实核查日志

| 序号 | 来源报告 | 被核查事实/数据 | 原始来源 | 核查结果 | 备注 |
|------|---------|---------------|---------|---------|------|
| 1 | XX-YY-{子专题}.md | [具体数据点] | [原始来源名称] | ✅ 准确 / ⚠️ 需修正 / ❌ 无法验证 | [说明] |
```

---

## 4. 审查不合格处理

- 明确指出问题所在及对应的责任专家
- 打回对应专家重新修改
- 修改后重新提交审查
- **终止条件**：单份文件最多打回 2 次（共 3 次提交机会）。达到终止条件后，在审查报告中记录遗留问题，继续推进后续流程

---

## 5. 领域适配指引

生成器在为具体任务构造 `{TASK}/rules/` 中的质量保证参考时，应在通用审查框架基础上补充：

- **该领域特有的质量标准**：如医学领域的循证等级、工程领域的可行性验证等
- **该领域的常见质量陷阱**：如该领域容易出现的数据偏差、逻辑谬误等
- **审查重点的领域调整**：根据事项特征调整三轮审查的具体关注点
