# L1 层共享规范

> 元声明
> - 版本：v1.0
> - 修订日期：2026-04-21
> - 实体层级：L1 元指令层
> - 用途：存放跨 L1 文件共同引用的规范，消除重复定义
> - 边界：本文件只存放被多个 L1 文件共同引用的通用规范，专项规范留在各自文件中

---

## 1. 层间隔离与自包含约束

每层自包含，不得引用其他层的文件。同层内文件可互相引用。

| 层级 | 引用范围 |
|------|---------|
| L1 层 | 仅引用 L1 层内文件（含本文件） |
| L2 层 | 仅引用 L2 层内文件，不得引用 L1 层 |
| L3 层 | 仅引用 L3 `{TASK}/` 内文件，不得引用 L1/L2 层 |
| L4 层 | 为最终研究成果，无引用需求 |

**L3 自包含要求**：生成器为每个任务动态生成的 `{TASK}/rules/` 必须使 L3 层完全自包含——引擎执行 AI 只需读取 `{TASK}/` 内的文件即可工作，无需访问 L2 层或 L1 层。

## 2. 内容实质性标准

引擎执行过程中产出的每份文件必须满足以下完成度标准，否则视为未完成。一份报告「完成」的标准不是字数到了，而是论证展开了。

### 2.1 完成度标准

**子专题研究报告的完成标准**：

1. **每个子发现都经历了完整的论证展开**——观察→解释→排他→边界，四个要素缺一不可。如果某个发现只到了「观察+解释」就停止了，该报告未完成
2. **分析与讨论不是研究发现的重述**——如果删除§4不影响读者对§3结论的理解，说明§4在凑字数而非论证
3. **论证的自足性测试**——将报告交给一个不了解该研究主题的读者，他能否仅凭报告内容（而非已有知识）理解每个核心结论为什么成立？如果不能，说明推理链有跳步
4. **至少引用 3 个明确的数据来源或文献**

**维度综合评估的完成标准**：

1. **跨子专题涌现的洞察**——综合评估的价值不在于总结每个子专题的结论（那是目录就能做到的事），而在于发现仅看单个子专题看不到的东西：子专题之间的印证、矛盾、或涌现模式
2. **必须逐一引用该维度下每份子专题的具体发现**

**扩展研究报告的完成标准**：

1. **跨维度综合产生了新的洞察**——不是把各维度的结论排列一遍，而是发现了维度之间的关联、冲突或涌现结构
2. **必须实际引用专题研究阶段的具体成果**

**综合判断报告的完成标准**：

1. **核心问题有明确回答**——不是「综合来看，建议……」这种模糊表述，而是直接回答事项的核心问题
2. **回答有推理支撑**——每个核心判断展示了证据整合的推理过程，而非简单罗列各维度的结论
3. **策略取舍有论证**——如果只能做三件事，做哪三件？为什么不做另外三件？
4. **必须引用前序阶段的具体数据和结论**

**最终报告的完成标准**：

1. **论证重建而非结论搬运**——读者仅阅读最终报告，即可理解从证据到核心结论的完整推理过程（详见 `L1-research-spec.md` §3.3）
2. **约束条件和不确定性不可省略**——子专题中识别出的约束、争议、不确定性必须在最终报告中保留

> 参考篇幅：以上完成标准通常需要子专题报告 3000-6000 字、维度综合评估 2000-4000 字、综合判断报告 4000-6000 字、最终报告 5000-10000 字。但篇幅是论证完整性的自然结果，不是目标——如果 2000 字就能完成全部论证展开，那 2000 字就足够；如果 6000 字还不够，就写到够为止。

### 2.2 认知质量要求

以下为定性判断标准，不宜量化为计数指标——否则会再次沦为可以不经思考就满足的官僚计量。

| 认知质量维度 | 要求 | 判别标准 |
|---|---|---|
| **推理链完整性** | 每个核心结论必须展示从证据到结论的推理路径 | 如果删掉结论只剩证据，读者能否自行推出同一结论？能，说明推理链完整；不能，说明有跳步 |
| **替代解释考量** | 对关键因果判断，必须考虑替代解释并说明取舍理由 | 如果对任何重要判断都只给出唯一解释，说明可能陷入确认偏误 |
| **实证与推测的区分** | 实证数据与估算推测必须明确标注，推测须说明依据和不确定性 | 如果估算值以「权威性评级：高」的表格呈现，说明在制造虚假精确感 |
| **论点递进** | 段落之间存在逻辑递进关系，而非并列罗列 | 如果打乱段落顺序不影响阅读，说明没有论点递进 |

**论证的操作定义**见 `L1-research-spec.md` §2.2，此处不重复。

**独立可读原则**：任何一份报告抽出来单独阅读，应能理解其研究问题和结论。

## 3. 数据源评估原则

### 3.1 权威性评级标准

| 评级 | 定义 | 典型来源 |
|------|------|---------|
| **高** | 经同行评审或权威机构发布，数据采集方法透明 | 顶级学术期刊、政府官方统计、国际组织报告（世界银行/IMF/WHO 等）、行业权威机构年报 |
| **中** | 来源可信但未经严格同行评审，或存在潜在利益偏向 | 知名咨询公司报告（麦肯锡/BCG 等）、行业协会数据、企业财报、主流媒体深度报道 |
| **低** | 来源不明、数据采集方法不透明、或存在明显偏向 | 个人博客、社交媒体、匿名来源、利益相关方的自我宣传材料 |

### 3.2 评级规则

- 默认使用高权威性来源，中权威性来源作为补充
- 低权威性来源仅在无替代来源时使用，且必须标注「来源权威性较低」
- 同一结论尽量由 2 个以上独立来源交叉验证

### 3.3 数据时效性要求

| 数据类型 | 推荐时效 | 说明 |
|---------|---------|------|
| 统计数据 | 最近 2 年 | 如只有更早数据，需标注「最新可得数据为 XXXX 年」 |
| 行业报告 | 最近 3 年 | 快速变化领域建议 1 年内 |
| 学术论文 | 最近 5 年 | 经典文献不受此限 |
| 政策法规 | 现行有效 | 需确认是否已被修订或废止 |

## 4. 术语定义

以下术语在 L1 层各文件中统一使用：

| 术语 | 定义 | 所在层级 |
|------|------|----------|
| 事项 | 用户输入的研究/分析/决策需求 | 全层 |
| 生成器 | L2 层提示词系统（`AGENTS.md`），负责接收事项并构造引擎 | L2 |
| 引擎实例 | 为特定事项构造的完整研究执行环境（`{TASK}/` 目录） | L3 |
| 维度 | 从事项属性推导出的独立研究视角 | L3 |
| 子专题 | 维度下的细分研究单元 | L3 |
| 专家智能体 | 负责特定维度或职能的 AI 角色定义 | L3 |
| 扩展研究 | 跨维度综合分析阶段（可选，由事项属性决定） | L3 |
| 阶段门控 | 进入下一工作流阶段前的强制验证检查 | L3 |
| 转化/适配 | 将 L1 规范定义转化为 L2 操作模板的过程 | L1→L2 |
| L2 `rules/` | 生成器层通用规范参考模板 | L2 |
| L3 `{TASK}/rules/` | 任务专属规范参考模板（由生成器动态生成） | L3 |
| 自包含 | 某层文件只引用本层内文件，不引用其他层 | 全层 |

## 5. 命名规范

目录和文件名统一使用**连字符**（`-`）作为分隔符，不使用下划线。此规范适用于 L1-L4 全部层级。
