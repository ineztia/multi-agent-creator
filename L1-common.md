# L1 层共享规范

> 元声明
> - 版本：v1.0
> - 修订日期：2026-04-21
> - 实体层级：L1 元指令层
> - 用途：存放跨 L1 文件共同引用的规范，消除重复定义
> - 边界：本文件只存放被多个 L1 文件共同引用的通用规范，专项规范留在各自文件中

---

## 1. 层间隔离与自包含约束

每层自包含，不得引用其他层的文件。同层内文件可互相引用。

| 层级 | 引用范围 |
|------|---------|
| L1 层 | 仅引用 L1 层内文件（含本文件） |
| L2 层 | 仅引用 L2 层内文件，不得引用 L1 层 |
| L3 层 | 仅引用 L3 `{TASK}/` 内文件，不得引用 L1/L2 层 |
| L4 层 | 为最终研究成果，无引用需求 |

**L3 自包含要求**：生成器为每个任务动态生成的 `{TASK}/rules/` 必须使 L3 层完全自包含——引擎执行 AI 只需读取 `{TASK}/` 内的文件即可工作，无需访问 L2 层或 L1 层。

---

## 2. 内容实质性标准

引擎执行过程中产出的每份文件必须满足以下最低要求，否则视为未完成：

| 产出类型 | 最低篇幅 | 必备章节 | 数据要求 |
|---------|---------|---------|---------|
| 子专题研究报告 | 2000 字正文 | 研究问题、研究方法、核心发现、分析论证、结论 | 至少引用 3 个明确的数据来源或文献 |
| 维度综合评估 | 2000 字正文 | 各子专题发现摘要、交叉分析、维度整体判断、关键洞察 | 必须逐一引用该维度下每份子专题的具体发现 |
| 扩展研究报告 | 2000 字正文 | 研究目标、跨维度分析、发现与洞察、结论 | 必须实际引用专题研究阶段的具体成果 |
| 综合判断报告 | 2000 字正文 | 成果整合、核心问题回答、趋势研判、建议 | 必须引用前序阶段的具体数据和结论 |

**独立可读原则**：任何一份报告抽出来单独阅读，应能理解其研究问题和结论。

## 3. 数据源评估原则

### 3.1 权威性评级标准

| 评级 | 定义 | 典型来源 |
|------|------|---------|
| **高** | 经同行评审或权威机构发布，数据采集方法透明 | 顶级学术期刊、政府官方统计、国际组织报告（世界银行/IMF/WHO 等）、行业权威机构年报 |
| **中** | 来源可信但未经严格同行评审，或存在潜在利益偏向 | 知名咨询公司报告（麦肯锡/BCG 等）、行业协会数据、企业财报、主流媒体深度报道 |
| **低** | 来源不明、数据采集方法不透明、或存在明显偏向 | 个人博客、社交媒体、匿名来源、利益相关方的自我宣传材料 |

### 3.2 评级规则

- 默认使用高权威性来源，中权威性来源作为补充
- 低权威性来源仅在无替代来源时使用，且必须标注「来源权威性较低」
- 同一结论尽量由 2 个以上独立来源交叉验证

### 3.3 数据时效性要求

| 数据类型 | 推荐时效 | 说明 |
|---------|---------|------|
| 统计数据 | 最近 2 年 | 如只有更早数据，需标注「最新可得数据为 XXXX 年」 |
| 行业报告 | 最近 3 年 | 快速变化领域建议 1 年内 |
| 学术论文 | 最近 5 年 | 经典文献不受此限 |
| 政策法规 | 现行有效 | 需确认是否已被修订或废止 |

## 4. 术语定义

以下术语在 L1 层各文件中统一使用：

| 术语 | 定义 | 所在层级 |
|------|------|----------|
| 事项 | 用户输入的研究/分析/决策需求 | 全层 |
| 生成器 | L2 层提示词系统（`AGENTS.md`），负责接收事项并构造引擎 | L2 |
| 引擎实例 | 为特定事项构造的完整研究执行环境（`{TASK}/` 目录） | L3 |
| 维度 | 从事项属性推导出的独立研究视角 | L3 |
| 子专题 | 维度下的细分研究单元 | L3 |
| 专家智能体 | 负责特定维度或职能的 AI 角色定义 | L3 |
| 扩展研究 | 跨维度综合分析阶段（可选，由事项属性决定） | L3 |
| 阶段门控 | 进入下一工作流阶段前的强制验证检查 | L3 |
| 转化/适配 | 将 L1 规范定义转化为 L2 操作模板的过程 | L1→L2 |
| L2 `rules/` | 生成器层通用规范参考模板 | L2 |
| L3 `{TASK}/rules/` | 任务专属规范参考模板（由生成器动态生成） | L3 |
| 自包含 | 某层文件只引用本层内文件，不引用其他层 | 全层 |

---

## 5. 命名规范

目录和文件名统一使用**连字符**（`-`）作为分隔符，不使用下划线。此规范适用于 L1-L4 全部层级。
