# 多元专家研究引擎生成器

一个基于大语言模型的智能研究引擎构造系统，接收任意事项并自动生成定制化的多元专家研究引擎。

## 四层架构

```
L1 元指令层（人类编写，指导构造 AI）
├── L1-agent-builder.md       # 元指令——定义构造过程
├── L1-agent-spec.md           # 生成器规范——定义生成器能力与方法论
├── L1-engine-spec.md          # 引擎规范——定义引擎结构与工作流
├── L1-research-spec.md        # 研究成果规范——定义成果标准与范例
└── L1-common.md               # 共享规范——层间隔离、内容标准、数据源评估、术语、命名

L2 生成器层（构造 AI 产出，供生成器 AI 加载）
├── AGENTS.md                  # 生成器入口提示词
└── rules/                     # 通用规范参考模板
    ├── engine-ref.md               # 引擎架构与执行纪律参考
    ├── research-writing-ref.md     # 研究成果写作规范参考
    ├── data-source-eval.md         # 数据源评估参考
    └── quality-assurance.md        # 质量保证参考

L3 研究引擎层（生成器产出，供引擎 AI 加载）
└── tasks/{UNIX时间戳}-{TASK}/
    ├── main.md                # 引擎入口与主路由
    ├── progress.md            # 执行进度追踪
    ├── agents/                # 专家智能体定义
    ├── rules/                 # 任务专属规范（从 L2 模板动态适配）
    └── XX-{维度}/             # 研究区目录

L4 研究成果层（引擎产出，供人类阅读）
└── tasks/{UNIX时间戳}-{TASK}/XX-*/*.md
```

**核心约束**：层间隔离——每层自包含，不得引用其他层的文件。

## 核心功能

- **事项分析**：接收自然语言描述的研究事项，分析其属性（含哲学审视）和复杂度
- **引擎生成**：为事项量身定制多维度专家研究引擎，一切由事项属性驱动
- **工作流规划**：五阶段工作流（专题研究→扩展研究→综合判断→质量审查→最终成稿）含阶段门控
- **专家体系**：配置维度专家、横向研究专家、综合判断专家、质量审查专家、报告撰写专家

## 工作原理

1. 用户输入研究事项（决策问题、科研课题、对比分析、趋势研判等）
2. 生成器分析事项属性，进行哲学审视，推导研究维度和专家配置
3. 按 5 步流程构造引擎实例（目录→main.md→专家定义→骨架→任务专属 rules/）
4. 另一个 AI 加载引擎逐文件串行执行研究，最终产出学术报告

## 使用要求

- 需接入大语言模型 API
- L1 层为规范定义（人类维护），L2 层为可执行系统（AI 构造），两层独立不交叉引用
